# **Модуль: vault_default_policy**

**Модуль vault_default_policy** – управляет встроенной политикой `default` в Vault. Vault автоматически прикрепляет эту политику к каждому токену, если явно не указано обратное через параметр `no_default_policy`. Модуль позволяет расширить базовый набор прав дополнительными правилами, которые автоматически получат все токены в кластере.

---

## **Как работает**

Политика `default` является единственной встроенной политикой Vault, которую нельзя удалить. Она содержит минимальный набор прав, необходимых для нормальной работы любого клиента: продление токена, проверка собственных прав, управление lease-ами. Vault применяет эту политику ко всем токенам автоматически, если только при выдаче токена явно не установлен флаг `no_default_policy = true`.

Модуль управляет этой политикой через Terraform. Шесть базовых правил присутствуют всегда и не могут быть удалены – они захардкожены в коде модуля. Через переменную `extra_rules` можно добавить дополнительные правила поверх базовых. Эти правила унаследуют абсолютно все токены в Vault, у которых не выключена политика `default`.

Ресурс создаётся с защитой `prevent_destroy = true`, поскольку Vault не позволяет удалить политику `default` – попытка это сделать через `terraform destroy` завершится ошибкой.

---

## **Параметры модуля**

| Параметр | Тип | Значение по умолчанию | Описание |
|---|---|---|---|
| `extra_rules` | `list(object)` | `[]` | Дополнительные правила поверх базовых. Структура: `path` + `capabilities` |

### Структура объекта в переменной `extra_rules`

| Поле | Тип | Описание |
|---|---|---|
| `path` | `string` | Vault path, поддерживает glob (`*`, `+`) |
| `capabilities` | `list(string)` | Список разрешений |

---

## **Базовые права (присутствуют всегда)**

Следующие правила являются частью модуля и применяются независимо от значения `extra_rules`:

| Path | Capabilities |
|---|---|
| `auth/token/lookup-self` | `read` |
| `auth/token/renew-self` | `update` |
| `auth/token/revoke-self` | `update` |
| `sys/leases/renew` | `update` |
| `sys/leases/lookup` | `update` |
| `sys/capabilities-self` | `update` |

---

## **Примеры вызова модуля**

### Простой вариант: только базовые права

Минимальный пример, воспроизводящий дефолтное поведение Vault:

```hcl
module "default_policy" {
  source = "../../../modules/vault_default_policy"
}
```

---

### Полный вариант: базовые права с расширением

Пример с добавлением права на чтение списка mount-ов для всех токенов:

```hcl
module "default_policy" {
  source = "../../../modules/vault_default_policy"

  extra_rules = [
    { path = "sys/mounts",       capabilities = ["read"] },
    { path = "sys/policies/acl", capabilities = ["list"] },
  ]
}
```

Итоговая политика будет содержать 6 базовых правил и 2 дополнительных. Права из `extra_rules` получат все токены в Vault, к которым прикреплена политика `default`.

---

## **Outputs**

После выполнения `terragrunt apply` модуль предоставляет следующие выходные значения:

| Output | Тип | Описание |
|---|---|---|
| `policy_name` | `string` | Имя политики (всегда `"default"`) |
| `base_rules` | `list(object)` | Список базовых правил, захардкоженных в модуле |

Запросить значения outputs можно с помощью следующих команд:

```bash
terragrunt output policy_name
terragrunt output -json base_rules
```

---

## **После apply**

После успешного применения конфигурации проверить итоговое содержимое политики можно с помощью следующих команд:

```bash
# Проверить итоговую политику (базовые + extra_rules)
vault policy read default

# Список всех политик
vault policy list
```

---

## **Важно**

Этот модуль рассчитан на **один экземпляр на весь Vault**. Не следует использовать его одновременно в нескольких стеках или окружениях – последний `apply` перезапишет предыдущий результат, что может привести к потере дополнительных правил.

Базовые права нельзя убрать или переопределить через `extra_rules`. Без `auth/token/renew-self` токены не смогут продлевать себя и будут терять доступ к Vault по истечении TTL. Без `auth/token/lookup-self` перестанет работать большинство клиентских библиотек. Эти права минимально необходимы для нормальной работы любого клиента Vault.

Примечание: Подробнее о том, как вызываются модули в рамках конкретного окружения, можно посмотреть в директории [`environments/`](../../environments/).
